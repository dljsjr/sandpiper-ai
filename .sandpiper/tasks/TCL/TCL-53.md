# Investigate project-level metadata and configuration

Investigate whether projects should carry their own metadata, configuration, or documentation beyond being container directories and namespaces.

Potential areas to explore:
- Project-level description/README (what is this project about?)
- Default field values per project (e.g., default reporter, default priority)
- Project-level labels/tags
- Project archival/active status
- Cross-project dependency policies
- Custom fields per project

The goal is to determine if project metadata would meaningfully improve the task management workflow, or if the current lightweight namespace approach is sufficient.
