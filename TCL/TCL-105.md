# storage migrate has no recovery path if interrupted mid-way

migrateInlineToSeparateBranch() uses a try/finally that deletes the tempDir backup on any error. If rmSync(workspacePath) succeeds but initSeparateBranch() throws (e.g. jj command fails), the original task files are destroyed.

Fix options:
1. Don't delete tempDir in the finally block on error — leave it in place so the user can recover manually
2. Only delete tempDir after cpSync(tempDir, workspacePath) succeeds
3. Document the recovery path in STORAGE.md (the user still has VCS history)
