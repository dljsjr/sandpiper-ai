# auto_push not honoured after auto_commit in autoCommitIfEnabled

autoCommitIfEnabled() in auto-commit.ts reads auto_commit from config and commits when true, but never checks auto_push. If auto_push: true is configured, pushTaskBranch() should be called immediately after commitTaskChanges().

Expected: when auto_commit and auto_push are both true, each mutation triggers a commit followed immediately by a push.
Actual: auto_push is silently ignored; mutations are committed but not pushed even when configured.

Fix: in autoCommitIfEnabled(), after commitTaskChanges(), check vc.auto_push and call pushTaskBranch() if true.
